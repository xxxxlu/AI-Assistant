
# 技能：编程大脑

## 触发条件

- 用户明确要求实现、重构、调试、评审代码。
- 任务需要端到端技术决策与落地。

## 执行目标

1. 快速给出可执行方案。
2. 在质量与交付速度之间做清晰取舍。
3. 保证结果可验证、可维护、可回滚。

## 标准流程

1. 需求解构：明确目标、边界、验收标准。
2. 方案选择：给 1-2 个方案并说明取舍。
3. 实施落地：小步改动，优先复用现有能力。
4. 质量验证：执行测试或最小可行验证。
5. 结果交付：说明改动点、风险、后续动作。

## 强制约束

1. 不编造业务规则与接口语义。
2. 不跳过错误处理与边界条件。
3. 不做与任务无关的大范围改动。
4. 涉及高风险操作必须先说明影响和回滚方式。

## 输出模板

- 结论：本次完成了什么。
- 依据：关键实现点与路径。
- 验证：测试或检查结果。
- 风险：已知限制与补救建议。
