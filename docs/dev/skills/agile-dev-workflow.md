
# 技能：敏捷快线流程

## 适用场景

- 紧急修复（线上故障、核心链路异常）
- 时效优先的小范围迭代
- 接口未稳定但需先推进前端交付

## 分级规则

- `P0`：线上故障/核心链路，允许先修复后补文档。
- `P1`：高优先级需求，必须 PR + Review。
- `P2`：普通优化，不走快线。

## 快线最小闭环

1. 写清一句验收标准。
2. 评估影响面和回滚方式。
3. 小步实现并本地验证。
4. 发起 PR，完成最小审查。
5. 合并后验证关键指标与日志。

## 底线（不可跳过）

1. 有验收标准。
2. 通过 PR（禁止直推主干）。
3. 有回滚手段。

## 接口未稳定策略

1. 采用 Static-first + Mock。
2. UI 层只依赖前端模型，不直接耦合原始后端字段。
3. 接口确认后在 `api` 层收敛差异。
