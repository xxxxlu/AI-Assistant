
# 用户偏好 (User Preferences)

## 0. 基本原则
1. **结果导向**: 结论先行，再给依据。
2. **小步快跑**: 任务拆分，频繁验证。
3. **可复用性**: 优先复用，其次重构，最后新建。


## 2. 工作流程 (Workflow)

### git 规范 (唯一口径)
- **分支**: main (受保护) | feature/* | bugfix/* | docs/*。
- **合并**: 必须 PR + 自测 + 记录回滚方案（针对 P0/P1）。

### 交付门禁
- **P0 (Critical)**: 验收标准 + PR + 回滚方案。
- **P1 (High)**: 方案说明 + PR + 测试记录。
