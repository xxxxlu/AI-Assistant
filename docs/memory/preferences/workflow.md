# 工作流程偏好

## 开发工作流

### 功能开发流程
1. **需求分析** - 理解需求，明确目标
2. **设计方案** - 技术选型，架构设计
3. **编码实现** - 遵循编码规范，小步迭代
4. **测试验证** - 单元测试，集成测试
5. **代码审查** - 自查 + AI 辅助审查
6. **文档更新** - 同步更新相关文档
7. **部署上线** - 发布和监控

### 任务优先级
1. **P0 - 紧急且重要** - 立即处理（如线上 bug）
2. **P1 - 重要不紧急** - 计划完成（核心功能）
3. **P2 - 紧急不重要** - 委托/延后（临时需求）
4. **P3 - 不紧急不重要** - 有时间再做（优化改进）

## Git 工作流

### 分支策略
```
main (生产)
  ├── develop (开发)
  │   ├── feature/user-auth (功能分支)
  │   ├── feature/doc-system
  │   └── bugfix/login-error
  └── hotfix/critical-bug (紧急修复)
```

### 分支命名规范
- `feature/功能名` - 新功能开发
- `bugfix/问题描述` - bug 修复
- `hotfix/紧急问题` - 生产环境紧急修复
- `refactor/重构内容` - 代码重构
- `docs/文档主题` - 文档更新

### 合并流程
1. 创建功能分支
2. 本地开发和测试
3. 提交代码，推送到远程
4. 发起 Pull Request
5. 代码审查（AI + 人工）
6. 合并到 develop
7. 定期合并到 main

## 时间管理

### 番茄工作法
- **工作时长**: 25 分钟专注
- **休息时间**: 5 分钟放松
- **长休息**: 4 个番茄后休息 15-30 分钟

### 一天的节奏
```
09:00-10:00  查看消息，规划任务
10:00-12:00  深度工作（核心开发）
12:00-13:30  午餐和休息
13:30-15:30  深度工作（功能实现）
15:30-16:00  休息和交流
16:00-18:00  测试、文档、代码审查
18:00-18:30  总结和规划明天
```

### 深度工作时段
- **上午 10:00-12:00** - 最高效，处理复杂问题
- **下午 14:00-16:00** - 次高效，功能实现
- **避免碎片化** - 关闭通知，专注当前任务

## 沟通偏好

### AI 辅助开发
- **代码生成**: 提供清晰的需求描述
- **代码审查**: 要求 AI 指出潜在问题
- **文档编写**: AI 帮助起草，人工精修
- **问题排查**: 分步骤描述问题，获取建议

### 提问方式
- **具体**: 提供上下文和代码示例
- **明确**: 说明期望的结果
- **结构化**: 使用列表、代码块等格式

### 反馈风格
- **建设性**: 指出问题并提供建议
- **及时**: 尽快给出反馈
- **清晰**: 明确说明改进方向

## 文档习惯

### 文档优先级
1. **README** - 项目入口，必须清晰
2. **架构文档** - 重要决策和设计
3. **API 文档** - 接口说明
4. **开发指南** - 环境搭建，开发规范
5. **更新日志** - 版本变更记录

### 文档更新时机
- **功能开发前**: 设计文档
- **功能完成后**: API 文档、使用文档
- **架构变更时**: 架构文档
- **定期回顾**: 每周/月检查文档更新

### 文档风格
- **简洁明了**: 避免冗长描述
- **示例丰富**: 提供代码示例
- **结构清晰**: 使用标题、列表、表格
- **持续更新**: 保持文档与代码同步

## 学习习惯

### 知识获取
- **官方文档优先**: 最准确可靠
- **实践验证**: 动手尝试，加深理解
- **记录笔记**: 整理成文档，方便复习
- **定期总结**: 周总结、月总结

### 技术选型
1. 阅读官方文档
2. 查看社区活跃度
3. 对比多个方案
4. 小范围试用
5. 记录决策原因（ADR）

### 问题解决
1. **定义问题**: 清晰描述问题
2. **搜索资料**: Google, Stack Overflow, GitHub Issues
3. **分析原因**: 定位问题根源
4. **尝试解决**: 逐步排查验证
5. **记录过程**: 写入错误日志或知识库

## 代码审查偏好

### 自查清单
- [ ] 代码风格符合规范
- [ ] 函数/变量命名清晰
- [ ] 有必要的注释
- [ ] 没有硬编码的值
- [ ] 错误处理完善
- [ ] 性能考虑合理
- [ ] 测试覆盖充分

### AI 审查重点
- 潜在的 bug
- 性能优化建议
- 安全漏洞
- 代码复杂度
- 最佳实践建议

## 休息和恢复

### 避免倦怠
- 定时休息，活动身体
- 保持工作生活平衡
- 适当运动和娱乐
- 充足睡眠（7-8 小时）

### 上下文切换
- 单次只专注一个任务
- 任务间留 5-10 分钟缓冲
- 记录当前进度，便于恢复
