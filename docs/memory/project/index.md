
# 项目上下文

## 作用

本目录存储项目长期稳定的事实信息，属于 L1（稳定层）记忆。

## 目录结构

| 文件 | 内容 |
|------|------|
| `brief.md` | 项目简介、目标、核心用户 |
| `architecture.md` | 系统架构、核心模块、数据流 |
| `tech-stack.md` | 技术栈、依赖、版本约定 |

## 使用场景

- 新人 onboarding
- AI 上下文初始化
- 技术选型评估
- 架构变更审查

## 维护规则

1. 仅记录已落地的事实，不记录计划
2. 架构变更后必须同步更新
3. 每月复核一次，确保一致性
4. 重大变更需创建 ADR
