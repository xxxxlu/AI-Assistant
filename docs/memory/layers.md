
# 记忆分层模型 (L1 / L2 / L3)

## 概览

记忆系统按稳定性和检索优先级分为三层，检索顺序 L1 → L2 → L3，冲突时高层级优先。

## L1 — 稳定层

长期有效、变更频率低的核心事实与规则。

| 目录 | 内容 | 维护频率 |
|------|------|----------|
| `memory/project/` | 项目简介、架构、技术栈 | 架构变更时 |
| `memory/preferences/` | Boss 可执行偏好 | 偏好变更时 |
| `memory/decisions/adr/` | 关键技术决策 | 新决策时 |
| `memory/auth/` | 身份与安全约束 | 每 90 天复核 |

**写入门槛**: 仅记录已落地的事实，不记录计划或推测。

## L2 — 经验层

可复用的短周期经验和阶段性复盘。

| 目录 | 内容 | 生命周期 |
|------|------|----------|
| `snippets/` | 场景-动作-结果碎片 | 按 GC 规则清理 |
| `retrospectives/` | 阶段性项目复盘 | 按 GC 规则清理 |
| `skills/` | 技能定义与工作流 | 长期维护 |

**写入门槛**: 一条记录解决一个问题；长文经验升级到 retrospectives。

## L3 — 外部层

本地知识库不足时的外部补全。

| 来源 | 优先级 | 约束 |
|------|--------|------|
| 官方文档 / 公告 | 最高 | 直接引用 |
| 一手仓库与发布页 | 高 | 标注来源 |
| 社区与媒体 | 中 | 必须标注不确定性 |

**使用规则**: 详见 [world/index.md](../world/index.md)。

## 检索流程

```
用户请求
  ↓
L1: memory/ (project, preferences, decisions, auth)
  ↓ 未命中
L2: snippets/ + retrospectives/ + skills/
  ↓ 未命中
L3: world/ + 外部搜索
```

## 冲突消解

1. 层级高者优先（L1 > L2 > L3）。
2. 同层级按 `last_reviewed` 或文件修改时间较新者优先。
3. 重复内容保留一份主文档，其余改为链接引用。
